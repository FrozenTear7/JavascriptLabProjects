EDYTOWANIE PROXY

Edycja proxy to metoda edycji, w której oryginalne klipy multimedialne są prezentowane na osi czasu za pomocą klipów proxy.

  • Racjonalne uzasadnienie
  • Ogólny obieg pracy dla proxy
  • Edycja proxy proxy Flowblade
  • Ważne uwagi dotyczące edycji proxy proxy Flowblade

Racjonalne uzasadnienie

Klipy proxy używane podczas edycji proxy mają zazwyczaj mniejszą szybkość transmisji danych i mniej intensywne dekodowanie procesora. Istnieją dwa główne powody używania edycji proxy:

  • Oryginalne nośniki, z których program jest edytowany, powodują zbyt wysokie zapotrzebowanie na przepustowość dysku lub moc obliczeniową procesora w celu dekodowania w celu umożliwienia elastycznej edycji
  • Oryginał przechowywany jest na serwerze sieciowym, wolny dysk zewnętrzny lub inny nośnik z ograniczonym dostępem i nie umożliwia szybkiej edycji w przypadku bezpośredniego dostępu.

Ogólny obieg pracy proxy

Wszystkie przepływy pracy przy edycji proxy mają te same fazy:

  1. Renderuj media proxy z oryginalnych nośników
  2. Zamień oryginalne nośniki na media proxy
  3. Edytuj za pomocą mediów proxy
  4. Zamień nośnik proxy na oryginalny nośnik
  5. Renderuj końcowy program przy użyciu oryginalnych mediów

Edycja proxy proxy Flowblade

  1. Tworzenie Media proxy
    1. Wybierz opcję Projekt-> Menedżer proxy w menu i ustaw ustawienia pliku proxy w obszarze Kodowanie Proxy
    2. Wybierz pliki wideo mediów w "Media" Panel
    3. Naciśnij przycisk z ikoną pliku proxy następnie przycisk Usuń w panelu "Media"
    4. Jeśli projekt jest już w trybie proxy Korzystanie z proxy proxy , klipy na osi czasu z oryginalnymi nośnikami, na których wyświetlono media proxy, zostaną natychmiast zastąpione przez media proxy
  2. Konwersja na nośnik proxy
    1. Wybierz opcję Projekt-> Menedżer proxy , aby otworzyć aplikację Proxy Manager
    2. Naciśnij przycisk Użyj Media proxy
  3. Edytowanie za pomocą mediów proxy
    1. Klipy korzystające z mediów proxy mają niebieski pasek wskazujący ten status
    2. Ikona wskaźnika edycji proxy jest wyświetlana w lewym dolnym rogu
    3. Jeśli nowe media proxy zostaną renderowane, klipy osi czasu z użyciem oryginalnego nośnika zostaną natychmiast zastąpione przez media proxy
    4. Projekt można zapisać normalnie i przekonwertować po wczytaniu na oryginalny nośnik
  4. Konwersja na oryginalne media
    1. Wybierz opcję Projekt-> Menedżer proxy , aby otworzyć aplikację Proxy Manager
    2. Naciśnij przycisk Użyj oryginalnego nośnika

Ważne uwagi dotyczące edycji proxy proxy Flowblade

  • ZNISZCZENIE JAKICHKOLWIEK MEDIÓW podczas edytowania proxy OCHRONA PRZED KORZYSTANIE Z ORYGINALNYCH MEDIÓW
  • Można używać tylko istniejących nośników proxy i klipów lub wszystkich oryginalnych nośników . Nie można używać tylko niektórych utworzonych mediów proxy.

    • Flowblade wykorzystuje technologię programistyczną, która zmienia ścieżki używane przez elementy multimedialne i klipy, aby wskazywać na ukryte media proxy lub oryginalne media.

     Przejście z jednego na drugie jest realizowane poprzez zapisanie ukrytego tymczasowego pliku projektu na dysku i zastąpienie ścieżek po odczytywaniu projektu. Z tego powodu brakujące oryginalne nośniki powodują, że konwersja powraca do użycia oryginalnego nośnika.
